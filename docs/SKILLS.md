# Catálogo de Skills

> **Status:** esqueleto inicial. As skills serão debatidas e aprovadas em DEC-08 / LEA-111.

## Objetivo

Padronizar procedimentos repetíveis do projeto. Skills não substituem memória nem decisões oficiais; elas definem como executar tipos específicos de trabalho.

## Estrutura padrão de uma Skill

Cada skill deverá conter:

1. identificador;
2. nome;
3. finalidade;
4. gatilhos de ativação;
5. pré-condições;
6. fontes obrigatórias;
7. procedimento;
8. restrições;
9. resultado esperado;
10. comandos disponíveis ao usuário.

## Limitação

Um arquivo Markdown não cria botões interativos nativos na interface do ChatGPT. Os comandos podem funcionar como opções textuais padronizadas e, quando houver suporte de interface, poderão ser associados a controles interativos.
