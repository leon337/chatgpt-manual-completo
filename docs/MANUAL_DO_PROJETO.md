# Manual do Projeto

> **Status:** estrutura inicial em rascunho.

## Finalidade

Este documento explica como planejar, produzir, revisar, aprovar, publicar e manter o Manual Completo da Plataforma ChatGPT.

## Fluxo operacional básico

```text
Linear identifica a próxima decisão ou entrega
        ↓
GitHub fornece o contexto oficial
        ↓
Documentação oficial confirma informações atuais
        ↓
ChatGPT produz planejamento ou rascunho
        ↓
Revisão e aprovação humana
        ↓
Publicação autorizada no GitHub
        ↓
Atualização do Linear e do PROJECT_STATE.md
```

## Consulta obrigatória

Antes de responder a solicitações relacionadas ao projeto, consultar:

1. `docs/CONSTITUICAO_DO_PROJETO.md`;
2. `PROJECT_STATE.md`;
3. `ROADMAP.md`;
4. `docs/SKILLS.md` quando houver procedimento aplicável;
5. issue ou projeto correspondente no Linear.

## Fonte de verdade

- **GitHub:** decisões, documentação e conteúdo aprovado/publicado.
- **Linear:** tarefas, prioridades, dependências, progresso e critérios de conclusão.
- **Chat:** ambiente de discussão, estudo e preparação de rascunhos.

## Regras temporárias

Enquanto a governança não estiver aprovada:

- tratar documentos como rascunho;
- não considerar uma decisão final apenas por estar escrita;
- registrar decisões no Linear;
- publicar somente quando houver autorização explícita;
- não excluir ou sobrescrever conteúdo sem autorização.
